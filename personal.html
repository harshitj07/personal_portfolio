ThermoChrg — Thermoelectric Portable Charger (Science Fair Project)
Tech: HTML, CSS (project site), Hardware Prototyping, Thermoelectrics
Timeline: 2021

ThermoChrg was a grade 8 science fair project where a friend and I designed and built a thermoelectric, wireless charging portable charger using gallium nitride (GaN) technology. The project explored how waste heat can be converted into usable electrical energy to power consumer devices.

What It Does:
- Converts thermal energy into electrical energy using thermoelectric principles
- Supports wireless charging for compatible devices
- Uses gallium nitride components to improve efficiency and reduce size

What it Matters:
- This project was my first exposure to applied engineering, showing how hardware innovation can be complemented by software and web-based documentation to communicate complex ideas effectively

Stock Market Prediction Using LSTM Neural Networks
Tech: Python, Machine Learning, LSTM Neural Networks
Timeline: 2024

This project focuses on predicting stock price trends using a Long Short-Term Memory (LSTM) neural network. Historical stock data for Apple Inc. (AAPL) was used to train and evaluate the model, highlighting both the potential and limitations of machine learning in financial forecasting.

What It Does:
- Collects and processes historical stock price data from Yahoo Finance
- Trains an LSTM model on over 10 years of price data
- Predicts stock price trends over a 60-day period
- Attempts short-term (next-day) price prediction for buy/sell insights

What I Learned:
- Implementing neural networks for time-series prediction
- Evaluating model performance using test vs real-world data
- Understanding the influence of external, non-quantitative factors
- Identifying improvements such as news integration and model comparison

Sorting Algorithms Benchmark + Visualizer
Tech: Java, Data Structures, Performance Benchmarking, Visualization
Timeline: 2025

A Java program that implements and compares multiple sorting algorithms by timing them on randomly generated arrays, printing results in a table, and visualizing at least two algorithms (including one O(n log n) method).

What I Learned:
- Translating algorithm theory into correct, testable implementations
- Measuring performance properly and interpreting results as input size scales
- Comparing O(n²) vs O(n log n) behavior with real timing data
- Building visualizations to make algorithm behavior understandable, not just fast
